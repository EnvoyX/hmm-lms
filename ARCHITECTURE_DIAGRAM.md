## Technology Stack

### Core Framework

- **Next.js 15** - React framework with App Router
- **TypeScript** - Type-safe development
- **tRPC** - End-to-end typesafe APIs

### Database & ORM

- **Prisma** - Next-generation ORM
- **PostgreSQL** - Primary database

### Authentication

- **NextAuth.js** - Authentication solution
- Credentials provider with email/password

### UI & Styling

- **Tailwind CSS** - Utility-first CSS framework
- **shadcn/ui** - Re-usable component library
- **TipTap** - Rich text editor

### Storage & Assets

- **AWS S3** - Object storage for files and media

### Development Tools

- **Bun** - Fast JavaScript runtime and package manager
- **Oxlint** - Code linting
- **Oxfmt** - Code formatting

## Data Flow

### Authentication Flow

```
User вҶ’ Login Form вҶ’ NextAuth Credentials Provider вҶ’
Verify Email/Password вҶ’ Create JWT Session вҶ’
Store in Database вҶ’ Return Session вҶ’ Client
```

### API Request Flow

```
Client Component вҶ’ tRPC Client Call вҶ’
tRPC Router вҶ’ Procedure (public/protected/admin) вҶ’
Context (session, db) вҶ’ Business Logic вҶ’
Prisma Query вҶ’ PostgreSQL вҶ’ Return Data вҶ’
Transform with SuperJSON вҶ’ Client
```

### File Upload Flow

```
User Uploads File вҶ’ Client Component вҶ’
tRPC API вҶ’ Generate S3 Presigned URL вҶ’
Direct Upload to AWS S3 вҶ’ Save Metadata to Database вҶ’
Return File URL вҶ’ Client
```

### Event RSVP Flow

```
User Views Event вҶ’ RSVP Form вҶ’
tRPC API вҶ’ Validate Form вҶ’ Create RSVP Response вҶ’
(If approval required) вҶ’ Admin Approval вҶ’
Google Sheets Sync вҶ’ Notification вҶ’ User
```

### Tryout Attempt Flow

```
User Starts Tryout вҶ’ Create UserAttempt вҶ’
Fetch Questions вҶ’ User Answers вҶ’
Submit Answers вҶ’ Calculate Score вҶ’
Save UserAnswer Records вҶ’ Update UserAttempt вҶ’
Return Results вҶ’ Analytics
```

## Role-Based Access Control

### Roles

- **STUDENT**: Access to courses, events, tryouts, forms
- **ADMIN**: Full access except user role management
- **SUPERADMIN**: Complete access including user role management
- **MACHINING**: Special role for machining-related content

### Procedure Protection

- `publicProcedure`: No authentication required
- `protectedProcedure`: Requires valid session
- `adminProcedure`: Requires ADMIN or SUPERADMIN role
- `superAdminProcedure`: Requires SUPERADMIN role only

## Security Features

- Role-based access control (RBAC)
- JWT session management with 7-day expiry
- Password hashing with bcrypt
- Prisma transaction isolation (Serializable)
- Input validation with Zod
- Type-safe API with tRPC
- Environment variable protection
- SQL injection prevention (Prisma ORM)
